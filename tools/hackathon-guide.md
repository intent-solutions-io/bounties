## How Hackathons Work

### Typical Flow
1. **Register** on Devpost before deadline
2. **Build** your project during the event window
3. **Submit** before deadline with:
   - Project description
   - Demo video (1-3 min)
   - GitHub repo link
   - Screenshots
4. **Wait** for judging
5. **Win** prizes if selected

### What Makes a Winning Project
- **Completeness:** Working demo > ambitious broken prototype
- **Polish:** Good UI/UX, clear video, clean code
- **Innovation:** Novel approach or unique solution
- **Impact:** Solves a real problem

### AI Tools Are Allowed
- Claude Code, Copilot, ChatGPT - all OK
- The judges care about the result, not how you typed it
- Vibe coding is totally acceptable
